## 2. 架构速览（接手必读）

### 2.1 通道：全中继 WebSocket
`控制端 ↔ lumen-server ↔ 被控端`，双方各连服务器，服务器**只解释控制面、盲转数据面**（`Relay` 载荷不透明
`serde_json::Value`，加新数据面帧服务端零改——前向兼容铁律，有单测守护）。**P2P 直连未做**（见 §3.3）。

### 2.2 数据面 = VT 字节流转发（SSH 魂）
被控端把每窗格 PTY 输出当裸 VT 字节转发，控制端喂进无 PTY 的镜像 `Terminal::advance` 复现。输入也是转发
VT 字节写被控端 PTY。**SSH 式「强制视口跟随」(`ViewportResize`) 已移除**（多会话下会扰动被控端别处 shell），
换成 `SubViewport`（后台 tab 1:1）+ 所有权规则。

### 2.3 关键协议帧（`crates/lumen-protocol/src/remote.rs`）
- 会话：`TabListSnapshot/TabCreated/TabClosed/TabUpdated`、`SubscribeSession`、`SubscriptionStarted`
  （携 panes 快照 + 行列权重 + maximized + focused）、`NewTab/CloseTab(+Result)`。
- 内容：`OutputWithId`、`ResizeWithId`（双 id 路由，`session_id` 非下标，**D1 铁律**）。
- 输入：`InputWithId{tab_id,session_id,data}`（旧 `Input` 休眠）。
- 回看：`HistoryReqForPane`/`HistoryRowsForPane`（带 session_id 校验；旧 `HistoryReq/Rows/Bounds` 单窗格用）。
- 尺寸/布局：`SubViewport`（绝对网格、后台 tab）、`SubLayout`（相对比例、双向、回声免疫）。
- 窗格操作：`PaneOp{tab_id,session_id,op}`，`PaneOpKind{New,Close,ToggleMaximize,SwapWith{other}}`
  （远程操作被控端，fire-and-forget，布局变化经 `SubscriptionStarted` 重发同步）。
- 文件：`RootChanged/ListDir(+Recursive)(+Result)/Fetch*/File*/Put*/MkDir*`（part3c-2）。

### 2.4 关键文件锚点
- `crates/lumen-protocol/src/remote.rs`：全部协议帧 + 往返测试。
- `crates/lumen-app/src/remote_ws.rs`：`RemoteWs` 状态机 + 帧收发/处理 + `run_connection` 读写循环
  （**`READ_TIMEOUT` 在 :62**）+ 镜像态（`mirror_panes`/`mirror_layout`/`mirror_active_pane`/`hist_*`）。
- `crates/lumen-app/src/main.rs`：App——被控端每帧推送/应用（`take_input`/`take_pane_ops`/`take_history_reqs`/
  `RootChanged` 推送）、控制端镜像渲染（多窗格离屏 1:1 + 回看 hist_term）、输入/滚轮/鼠标路由。
- `crates/lumen-app/src/shell/mod.rs`：镜像 UI（`remote_mirror_multi` 块：标题栏 + ✕/最大化按钮 + 拖动换位
  + 可拖分隔条 + 焦点高亮）；本地窗格交互在镜像覆盖时被抑制。
- `server/lumen-server/`：中继（`hub.rs` 会话/配对独占、`ws.rs` 读循环、`handlers.rs` REST、`config.rs` 配置）。

## 3. 待办（三大块，按性质独立）

### 3.1 ⏳ 健壮性收尾（内网即可受益）
1. **文件服务线程统一**（review MED-1，`remote_ws.rs:~1812`）：现每读盘请求起一条 OS 线程、无并发上限；
   慢盘大量展开会堆线程。收敛为「常驻 service 线程 + 有界任务队列 + inflight 上限」。
2. **输入彻底零延迟**：现 `READ_TIMEOUT=5ms` 轮询已基本无感；要 0 延迟需 `run_connection` **读写分两线程**
   （tungstenite 单 socket 拆分有工程量：可 `TcpStream::try_clone` 全双工 + 写线程 `cmd_rx.recv()` 阻塞即写）。
3. **文件传输 UI 收尾**（M5.4 原计划、当前未做）：双击打开远程文件→落临时夹 + **LRU 容量淘汰**
   （`paths.rs` 加 `temp_dir()`，默认上限 500M 可调）；**状态栏传输进度**（`shell/statusbar.rs` 中间区：
   文件名 + 进度条 + 并发计数，多文件每几秒轮换）。

### 3.2 ❌ 对外安全硬化 + 生产部署（上公网前必做，M5.1 安全审查已列）
> 当前全是**局域网/本地 dev**：明文 `ws://`、默认 JWT 密钥、本地 Docker Postgres。
1. **强制 TLS**：Caddy 反代 + Let's Encrypt 自动签发/续期；客户端走 `wss://`（`rustls-tls-native-roots` 已支持）。
2. **云端 Linux 部署**：`cargo build -p lumen-server`（Linux 版，纯跨平台 crate）→ VPS（已有域名）+ 云 Postgres；
   国内需 **ICP 备案**。
3. **认证端点限流 / 防撞库**（登录/注册）。
4. **token 服务端可吊销**（`token_version` 字段，登出/换密即失效）。
5. **客户端 token 加密存储**（Windows **DPAPI**，现明文）。
6. **登录端点账户枚举收口**（未知账号与错密码返回不可区分）。
7. **历史分页 `has_more` 信号**。
8. **生产 JWT 强密钥**（`LUMEN_JWT_SECRET`，现 dev 默认密钥仅回环/局域网告警）。

### 3.3 ❌ P2P 直连（M5 之后的大优化项，设计时即推后）
> 设计原话（`架构设计.md:125`、`M5远程控制设计.md` Q1/§8）：**中继优先 + P2P 终局；P2P 直连（QUIC 打洞
> + 中继回退）作为 M5 之后的优化项，M5 全中继跑通、形态稳定后再做。** 现在形态已稳定，可以开做。
- **目标**：控制端↔被控端**直连**，绕开服务器中继 → 降延迟（省一跳，接续刚修的输入延迟的「终极版」）+ 降
  服务器带宽/成本（文件传输尤其受益）。
- **技术路线**：**QUIC 打洞 + 中继回退**。
  1. **信令**：仍用中继 server 牵线（交换公网端点候选）。
  2. **NAT 穿透**：QUIC 打洞（双方同时向对端公网端点发包穿透）。
  3. **直连数据面**：打通后输入/输出/文件直接走 P2P；数据面帧复用现有 `RemoteFrame`（传输层换掉即可）。
  4. **中继回退**：对称 NAT 等打洞失败 → 自动回退全中继（**中继这套不废，是 P2P 兜底**）。
  5. **E2E 加密**：直连通道端到端加密（不经服务器，安全模型变化，需重新审查）。
- **工作量**：独立大项。引入 QUIC 栈（如 `quinn`）、打洞协调协议、连接状态机（直连/回退切换）、E2E 密钥协商。

### 3.4 建议优先级
- **只内网用** → 做 3.1（健壮性）即可。
- **上公网给真实用户** → 先 3.2（安全+部署，缺 TLS/限流/token 吊销直接上线有风险），再 3.3（P2P 优化）。

## 4. 协作铁律 & 坑（接手必看）

1. **构建交付铁律**：`cargo build --bin lumen` 产 `target/debug/lumen.exe`；海风哥实际运行的是同目录
   **`lumen-debug.exe`**（手工重命名约定）。**每次构建后必须** `Copy-Item lumen.exe → lumen-debug.exe -Force`
   + 哈希核对一致再交付。漏这步=他重启到旧二进制、白跑一轮（已踩过）。
2. **两机部署**：**协议改动**（加/改 `RemoteFrame` 变体）→ **两端都刷**；纯控制端改动只刷本机。覆盖前若
   `lumen-debug.exe` 在跑，**只 `Get-Process lumen-debug | Stop-Process`**（别按 `lumen` 名 kill，会误杀）。
3. **server 启动**：① Docker `lumen-postgres`（host 5544→容器 5432）② `target/debug/lumen-server.exe`
   （绑 `0.0.0.0:8787`）。控制端连 `http://127.0.0.1:8787`；被控端(Test1)连**本机局域网 IP**（如
   `http://192.168.1.85:8787`，别填 172.x Docker/WSL 虚拟网卡）。server 挂在会话下时**会话重启即被收走**，
   要常驻得独立窗口跑。
4. **质量门**：`cargo clippy -p lumen-protocol -p lumen-app --all-targets -- -D warnings` 0 警告；
   `cargo test` 三连绿（当前 lumen-app 446 + lumen-protocol 18）；禁 unwrap、unsafe 带 SAFETY。
5. **延迟坑**：`READ_TIMEOUT`（`remote_ws.rs:62`）既是读超时**也是出站延迟上限**（单线程循环读阻塞期间发不出
   出站）。改大会让打字卡顿。
6. **D1 路由铁律**：`(tab_id, session_id)` 双 id 路由，**绝不用窗格下标做路由/缓存键**（关窗格令 `Vec` 重排）。
7. **最大化契约**：`toggle_maximize_pane(ti,pi)` 的「补帧三件套·勿拆」（清 maximized + 全窗格超龄
   term_frame_due + request_redraw）——见 `main.rs` 注释，新增「帧内清 maximized」动作须复制这套。
8. **回声免疫**：`SubLayout` 双向同步靠 `sub_layout_baseline`（收到对端比例即更新基线，应用不再回发），
   连续拖动两向无打架。
9. **测试需两机肉眼验收**：镜像视觉/输入/选区/IME/换位等无法纯自测；改动后用两机过验收清单。
