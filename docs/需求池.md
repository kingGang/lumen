### P1 蓝线闪烁诊断结论（2026-06-11）

- **根因**：命令块左缘「运行中」蓝色状态条（`lumen-renderer/src/lib.rs:425`，色值 Tokyo Night ANSI 蓝 `#7aa2f7`，绘制于 `lib.rs:408` 的 `if !term.is_alt_screen()` 分支）。codex/Claude Code CLI 跑在**主屏**（非 alt screen，用 ANSI 光标移动 + DEC 2026 同步批做行内重绘），始终是一个未闭合的「运行中」块 → 左缘常驻蓝条。
- **闪烁机理**：codex 持续输出致主屏滚动，蓝条跟随块可见行区间移动；昨夜渲染门控（`main.rs` `skip_term_render`：同步区间跳过离屏渲染、保留上一帧）令蓝条位置更新与补帧不连续 → 视觉闪烁。光标因有独立冻结防抖（`cursor_frozen_at`）跳帧时整体不画，故不闪。
- **候选 2/3 排除**：侧栏激活条（`bg_highlight #292e42`，蓝灰非纯蓝、且与终端区物理分离）、文件树交界（露底色为深灰黑非蓝）均不符。
- **修复方向（放打磨批次，碰渲染门控核心需一起回归）**：①跳帧时仍消费 `take_dirty()` 并对块矩形几何单独同步；或 ②ESU 快路直渲永不跳帧、门控只作用于静默合帧定时路径（更简单，倾向此方案）。需 LUMEN_PERF 验证修后空闲仍零渲染、无空转回退。
- **海风哥已确认（2026-06-11）**：蓝线就在 codex 文字区域最左列——候选 1 坐实。

### P8 双击失效诊断结论（2026-06-11）

- **根因**：egui_ltreeview 0.7.0 上游 bug——Click 处理里单槽 output 先置
  `SetLastclicked` 随即被 `SelectOneNode` 无条件覆盖，`last_clicked_node`
  永不记录 → 双击判定 `was_clicked_last` 恒 false → 鼠标双击的
  `Action::Activate` 不可达（回车键独立路径不受影响）。自 M3.3 引入
  0.7.0 起即存在（当时验收走的回车/目录场景未暴露），**与 M3.6 拖拽
  改造无关**（egui 0.34 下 click_and_drag 的干净双击不会进入拖拽分支，
  已读源码核对）。crates.io 最新仍是 0.7.0，无升级可用。
- **排除项**：打开调用本身经函数级验证无问题——`explorer.exe <路径>`
  对裸路径与含空格/中文的引号路径均正确调起关联程序（spawn Ok +
  关联编辑器窗口标题核对）；搜索结果列表的双击走 egui 原生
  `double_clicked()`，本来就不经 ltreeview，无此 bug。
- **修复**：`merge_double_click_activation` 合成激活（整树 Response 双击
  + 本帧 SetSelected → 激活点选节点），真实 Activate 优先、上游修复后
  自动让位；附无头 egui 复现单测作上游行为哨兵（上游修复时测试失败
  提醒移除合成分支）。
