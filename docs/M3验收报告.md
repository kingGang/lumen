## 二、夜间深度审查与修复

对 M3 全部改动跑了 5 维度多 agent 审查（多会话并发 / GPU 资源 / 输入 IME 路由 / 数据 IO 边界 / UI 状态一致性），21 个发现经逐项对抗验证确认 18 项（去重后 14 个独立问题），已全部修复（`1148359` + `7170919`）并回归。重点几个：

| 级别 | 问题 | 修复 |
|---|---|---|
| high | 鼠标划过窗口触发的重绘绕过 DEC 2026 同步门控——codex/vim 重绘期间会把半成品帧画上屏，且无 8ms 限频（1000Hz 鼠标可打满主线程） | 同步区间跳过终端渲染（保留上一完整帧）+ 事件驱动重绘并入 8ms 合帧调度 + 终端/UI 时间戳分离；实测 egui 启动动画从 ~2500fps 降到 8.4ms/帧，空闲零渲染 |
| high | 文件树 cd 注入：PowerShell 把 Unicode 弯引号（U+2018/2019/201B）当单引号，目录名含弯引号可逃逸字符串执行任意命令 | 同形字全集翻倍转义 + 控制字符路径直接拒绝（纵深防御），补危险字符单测 |
| medium | 跨 DPI 显示器拖动后字号失真（ScaleFactorChanged 未处理） | 补分支 + renderer 可重入缩放接口 |
| medium | 文件树读目录在 UI 线程同步阻塞——无响应网络盘会冻死整个应用 | 后台线程读盘 + 加载中占位 + 代次号防旧根污染 |
| medium | 右键菜单/重命名/覆盖层的多处焦点仲裁漏洞（按键可泄漏进 shell） | 统一焦点归属计算，菜单开启键盘归 egui |
| low×9 | 拖选状态残留、IME 不回滚、Ctrl+, 开关不对称、配置整份连坐降级、滑块高频写盘、分数 DPI 接缝、BOM 误降级等 | 全部修复，新增单测 10 个 |

审查驳回 3 项（对抗验证确认不成立）。**遗留一项架构级改进**未动：后台会话洪泛时存在最多约 10-30ms 的回显延迟尖峰（全局通道队头阻塞），方案已记录（per-session 通道 + 优先轮询激活会话），建议 M4 前单独立项。

## 三、留给海风哥的人工验收清单（环境拿不到屏幕，视觉与 IME 需真人）

1. **整体观感**：三栏布局 + 顶栏头像，对照 `docs/截图/warp界面1.png` 看风格差距
2. **中文输入法**：终端打中文；点侧栏/设置页再点回终端打中文；vim 里打中文（egui 偷关 IME 是头号风险，已做每帧复位）
3. **多会话**：Ctrl+T 开 3 个 tab 各跑各的；一个 tab `ping -t` 切走再切回看未读点；vim 里 Ctrl+W 不能关 tab
4. **文件树**：cd 后树根跟随；双击目录/文件；长命令运行中双击目录不应注入；Ctrl+B 折叠后排版正常
5. **设置**：Ctrl+, 改字号立即生效、重启保留；拖滑块只在松手时生效；浅色主题看一眼（色板未经设计调校）
6. **登录**：头像菜单 → Log in → 任意邮箱+密码 → 头像变首字母 → 重启保持 → Log out 回占位
7. **codex/TUI 手感**：跑 codex 快速打字 + 同时晃鼠标——不应见闪烁/残行（夜间修的 high 级渲染门控就是为这个场景）
