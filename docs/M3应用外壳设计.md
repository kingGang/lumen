# Lumen M3「应用外壳」设计文档

> 状态：定稿（2026-06-10，基于两路技术调研综合，调研全文见 `docs/调研/`）
> 前置：M1 终端底座、M2 Blocks UI 已完成。本文档对应 `docs/架构设计.md` §4 里程碑 M3。
> 范围（海风哥拍板，五项，界面对标 Warp，参考截图 `docs/截图/`）：①最左侧栏 tab 列表 ②中间一栏文件树 ③右侧命令行工作区 ④右上角登录头像 + 登录界面 ⑤设置界面。
> 已定决策：先外壳后输入编辑器；UI 用 egui；文件树只读浏览 + 跟随 cwd；登录纯 UI + 本地假登录（真后端 M5 随远程控制上线）。

---

## 1. 总体架构

```
winit ApplicationHandler（现有事件循环保留）
  │ window_event ──▶ egui_winit::State::on_window_event ──▶ EventResponse{consumed}
  │                      │ consumed=false（终端聚焦）→ 现有键盘/IME/鼠标 → PTY 管线
  ▼
egui::Context::run（每可见帧）
  ├─ 左侧栏：tab 列表（多会话）
  ├─ 中间栏：文件树（egui_ltreeview，可折叠隐藏）
  ├─ 顶部右：头像/登录入口；弹层：设置页、登录页、头像菜单
  └─ CentralPanel(Frame::NONE)：ui.image(终端离屏纹理) ← 终端管线原样渲染到 offscreen texture
                                                            （register_native_texture 注册）
```

三条核心决策：

1. **egui 主导布局，终端走离屏纹理嵌入**（裁决自两份调研，采用 `docs/调研/M3-egui集成调研.md` 方案 b）：现有 lumen-renderer 管线**原封不动**（自管 pass、damage 部分重绘、自有 atlas），仅渲染目标从 surface 换成持久 offscreen texture；egui 侧 `register_native_texture` 拿 `TextureId`，工作区当 `ui.image()` 画。弹窗/菜单/设置页盖在终端上 z-order 天然正确；终端区 resize 时重建纹理 + `update_egui_texture_from_wgpu_texture` 原地换绑。
2. **版本锁**：egui/egui-wgpu/egui-winit 统一 0.34.3（实查依赖 wgpu 29.0.1 / winit 0.30.13，与现有栈一致）；wgpu 类型经 lumen-renderer `pub use wgpu` 取用，保证同版本同类型。
3. **多会话是外壳的数据地基**：左侧 tab 列表背后是 `Vec<Session>` 重构（见 §2），先于 UI 落地。

## 2. 多会话架构（单会话 → `Vec<Session>`）

依据 `docs/调研/M3-多会话与egui改造面评估.md`（含逐字段行号清单），要点：

- **Session 持有**：`term`（含 grid/scrollback/blocks/滚动/标题）、`pty`、`last_esu_mark`、光标防抖（`cursor_displayed`/`cursor_frozen_at`）、选区与 `selected_block`、`redraw_*` 渲染计划、`has_unseen_output`（tab 未读点）。
- **全局保留**：window、renderer、`wake_pending`、modifiers、clipboard、`mouse_pos`、perf 埋点、`last_render_at`（8ms 合帧基准）；新增 `sessions: Vec<Session>`、`active: usize`、`next_session_id`。
- **通道改造**：per-session 转发线程保留，汇入**同一条全局通道**，事件变 `(SessionId, PtyEvent)`；`EventLoop<PtyWake>` user event 保持无数据、`wake_pending` 全局去重协议零变化。drain 时按 sid 路由，已关闭会话的残留数据直接丢弃。
- **后台会话消化不渲染**：drain + `term.advance` + `take_responses` 回写对**所有**会话照常执行（DSR/DA 不回写会卡死后台程序）；渲染调度段加 `sid == active` 闸门，后台只更新自己的 `last_esu_mark` 并标记未读。每次 drain 给后台会话设字节上限，防 `yes` 级输出抢占主线程。
- **resize 对全部会话生效**：所有 tab 共享同一终端视口矩形（来自 egui 布局），变化时逐会话 `term.resize + pty.resize`，不做懒 resize（防切换瞬间花屏）。
- **生命周期**：`PtyEvent::Exited` 从「退出进程」改为「关该 tab」，最后一个 tab 关闭才退出；切 tab 时清理 `cursor_frozen_at`/`redraw_*` 残留并立即 request_redraw。

### 2.1 分屏（F5 / M3.7 追加）

- **数据模型**升级为 `Vec<Tab>`，每 tab 1~6 个窗格（`Tab { custom_title, panes: Vec<Session>, focused }`）；原「激活会话」语义 = 激活 tab 的**焦点窗格**（窗口标题、IME、键盘/滚轮/选区/粘贴、文件树 cwd 全部跟随）。drain 按「焦点窗格 → 激活 tab 其余窗格（限额）→ 后台 tab」轮询。
- **布局**固定均分（`shell/layout.rs` 纯函数）：1=满屏 2=左右 3=左中右 4=上2下2 5=上3下2 6=上3下3，两排时上排在前；窗格各自一张离屏纹理（renderer 按会话 id 管理 offscreen 与行排版缓存），同步门控/resize/光标防抖逐窗格独立。
- **交互**：点击窗格聚焦（多窗格画 1px accent 边框）；顶栏右侧「＋」/ Ctrl+Shift+D 新增窗格（满 6 禁用），悬停窗格右上角 ✕ / Ctrl+Shift+W / shell exit 关闭窗格（最后一格 = 关 tab）；文件树拖放以**落点窗格**为插入目标（先聚焦再插入），双击目录 cd 作用于焦点窗格；侧栏多窗格 tab 标「N 格」。
- **持久化** sessions.json 升级 v2 嵌套结构 `{version: 2, active_tab, tabs: [{custom_title, focused, panes: [{cwd}]}]}`；v1 平铺（M3.6b）与批1 过渡格式读侧自动迁移；重启还原窗格数量、布局（由数量推导）、各窗格 cwd、焦点位与激活 tab（失效 cwd 回退默认 + toast 一次，单窗格 spawn 失败跳过不连坐）。
- **比例布局与窗格标题栏**（F7①③ + P11 / M3.7c 批1 追加）：布局升级可调比例（`PaneLayout` 排高/列宽权重挂 Tab，1~6 格网格结构规则不变），窗格间 1 物理像素灰阶分隔条 = 可视线 + 拖动手柄一体（拖动按指针绝对位置调相邻两格比例、最小 120×80 逻辑 px 钳制、双击恢复均分），权重随 sessions.json 持久化（`row_weights`/`col_weights` 可选字段，旧文件双向兼容）；每窗格顶部 24px 标题栏（cwd 尾目录名 > OSC 标题 > 「窗格 N」，✕ 常驻右端，单格也显示），`pane_rects` 输出**内容矩形**（标题栏占高从终端扣除，标题栏鼠标事件不进终端）。
- **拖动标题栏换位与侧栏拖宽**（F7② + P10 / M3.7c 批2 追加）：标题栏即抓手（egui 点击/拖动阈值仲裁，按下移动超 6 逻辑 px 才算拖），拖动中目标窗格 accent 高亮（2px 描边 + 8% 蒙层）+ 指针旁半透明标题小卡跟手；松手落在其他窗格 = 交换两窗格**内容**（panes 下标互换、布局权重不动），焦点跟随被拖窗格落位、即时落盘，落在源格/窗格外取消无副作用。左侧 tab 栏（140~320 逻辑 px）与文件树栏（160~480）可拖宽，宽度松手落盘 settings.json `layout` 节、重启还原；拖宽手柄与分隔条同款 raw 鼠标让位（不抢终端焦点）。

## 3. egui 集成关键点

### 3.1 事件与焦点路由

- `window_event` 入口先喂 `on_window_event`；`consumed=false` 时走现有终端输入管线。`Resized`/`CloseRequested`/`RedrawRequested` 永远自己处理。
- 自建**终端聚焦布尔**（点击终端区获焦 / 点击 egui 面板失焦）——egui 不会为非控件区域持焦点，所有键盘与 IME 路由依赖它。
- Tab 键：终端聚焦时绕过 egui 焦点导航，直通 PTY。
- 终端区鼠标（选区/块点击/滚轮）以 `!consumed` 为闸；`cell_at` 坐标改为相对终端区矩形。

### 3.2 IME 三坑与对策（Windows 中文输入是硬验收）

| 坑 | 对策 |
|---|---|
| egui-winit 按自己的文本焦点调 `set_ime_allowed(false)`，关掉整窗 IME，终端中文失效 | 每帧 `handle_platform_output` **之后**，终端聚焦则强制 `set_ime_allowed(true)` + `set_ime_cursor_area(终端光标矩形)` |
| `Ime::Commit` 现在无条件写 PTY，egui 输入框聚焦时会双投 | 按 `wants_keyboard_input()` 路由：egui 聚焦则不写 PTY |
| 候选框停在 egui 上次设置的位置 | 终端聚焦时按光标格子坐标（终端区原点 + cell×cursor）设置 cursor area，且在 egui 之后调用覆盖 |

另：宿主可设 `ImePurpose::Terminal`；Commit 后紧跟的 KeyboardInput 重复注入沿用 M1 已有处理。

### 3.3 渲染与节拍

- 帧流程：acquire → egui pass（`LoadOp::Clear`，内含终端纹理采样 + 全部 widget）→ present；`update_buffers` 先于 `render`，`textures_delta.free` 放 render 之后（集成骨架见调研文档 §4）。
- PTY 输出 → 更新离屏纹理 + request_redraw；ESU 直渲快路语义不变；`about_to_wait` 里 egui `repaint_delay` 与现有 `redraw_at` 取 min，**严守 ControlFlow 复位约定**（历史上有空转事故）。
- 每个可见帧多一次 egui 布局（约 0.1-1ms）：用 LUMEN_PERF 埋点对比 M2 基线，打字延迟回归是 M3.1 验收项。

### 3.4 主题与字体

- Warp 风深色全部静态配置可达（`panel_fill`/`CornerRadius`/`Frame` 逐面板覆盖，配色样例见调研文档 §3）；色板从 `docs/截图/` 取色，与终端区 Tokyo Night 主题统一。
- 中文字体：`FontData::from_owned(读取 C:/Windows/Fonts/msyh.ttc)` + `index` 选字面，插入 fallback 列表；运行时读取（19MB 不进二进制）。

### 3.5 文件树

- 采用 `egui_ltreeview 0.7.0`（egui 0.34 配套，键盘导航/选中/激活/右键菜单齐全；已知短板：单一维护者、无虚拟化——按当前目录**懒加载子层**规避）。
- 跟随 cwd：依赖 OSC 9;9 cwd 上报——把输入编辑器设计 P0 中的「integration.ps1 上报 + lumen-term `cwd` 字段」**提前到本期 M3.3 实现**（M4 直接受益）。
- 交互（已拍板只读起步）：点目录 → 激活会话执行 cd；点文件 → 系统默认程序打开；不做内置查看/编辑。

## 5. 实施分期与验收

| 期 | 范围 | 验收 | 估时 |
|---|---|---|---|
| **M3.1 egui 底座** | egui 三件套接入、终端离屏纹理嵌入、事件/IME 路由与终端聚焦模型、主题+中文字体、空壳侧栏占位 | M1/M2 全行为零回归（手测清单：ls/git log/vim/codex/**中文输入**/粘贴/选区/块操作）；打字延迟埋点对比不劣化；clippy 零告警 | ~3 天 |
| **M3.2 多会话 + tab 列表** | `Vec<Session>` 重构、通道带 id、tab 新建/关闭/切换/未读点/快捷键 | 双 tab 各跑任务互不干扰；后台 tab 跑 `ping -t` 前台打字不卡；关 tab 进程随杀；切换 <100ms 无花屏 | ~3 天 |
| **M3.3 文件树** | OSC 9;9 cwd 链路（脚本+term 字段）、ltreeview 懒加载、点目录 cd/点文件系统打开、折叠 | `cd` 后树根跟随；万级文件目录不卡；隐藏目录默认不展示 | ~1.5 天 |
| **M3.4 设置界面** | 分类导航+内容区、Appearance 立即生效、settings.json 持久化与启动加载 | 改字号/主题即时生效且重启保留；设置页开着终端照常输出 | ~2 天 |
| **M3.5 登录 UI** | 登录卡片、mock 账号、头像与下拉菜单 | 登录→头像变化→重启保持→登出回占位，全流程顺畅 | ~1.5 天 |

> 合计约 11 人日。每期跑 M2 手测回归（重点中文 IME），`cargo clippy -- -D warnings` 零告警；**不开 LTO**（本机构建铁律）。

## 6. 风险清单

| # | 风险 | 缓解 |
|---|---|---|
| 1 | **IME `set_ime_allowed` 被 egui 反复翻转**（最高风险） | M3.1 第一天专项实测复位时序；每帧终端聚焦即强制复位；微软拼音 + 搜狗双输入法回归 |
| 2 | 打字延迟回归（每帧叠加 egui 布局） | LUMEN_PERF 埋点 M2 基线对比；超标则裁剪 egui 帧内容（侧栏静态化） |
| 3 | ControlFlow 粘性陷阱重现（egui repaint_delay 合流） | 沿用 main.rs 既有复位约定；空转监控埋点 |
| 4 | Tab/方向键被 egui 焦点导航偷走 | 终端聚焦时绕过 egui 键盘事件分发，单测语义表 |
| 5 | 后台会话 `advance` 抢主线程 | 后台 drain 字节上限，剩余下个 wake 再消化 |
| 6 | egui_ltreeview 单一维护者 | 用法收敛在 filetree 模块内，必要时可替换为自绘 |
| 7 | srgb/半透明叠加颜色偏差（egui 与现有 rect 管线） | M3.1 目检选中块 tint 与弹层阴影 |
| 8 | 多会话重构破坏 M2 行为 | 重构纯机械（字段搬家），分两个 commit：先搬结构后接 UI；每步跑回归 |

## 7. 不做清单（本期防蔓延）

真账号后端（M5）、文件树虚拟化/内置编辑器、tab 拖拽排序、分屏（split pane）、会话持久化恢复、自定义快捷键编辑、多窗口、毛玻璃/渐变视觉、设置同步（Settings sync 开关仅 UI 占位）——有真实痛点再立项。（后注：分屏与会话持久化已分别以 F5/M3.7、F4/M3.6b 立项交付，见 §2.1 与需求池。）
