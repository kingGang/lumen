## M5.3 终端远程（海风哥已拍板；part1 已落地）

- **形态**：选中在线设备 → 控制端**镜像被控端整个编辑器实例**（会话 tab/目录树/终端），同 Claude Code 远程会话。**状态增量同步 + 操作指令（非画面流）**——见设计 §5.2。
- **通道**：**WebSocket 长连接经 lumen-server 中继**（M5.2 只用 HTTP 心跳）。生产 TLS 由 Caddy 反代终止。P2P 留后续。
- **连接授权**：首次连接 **9 位数字配对码**；被控端醒目「正在被远程控制」指示 + 一键断开。
- **并发/仲裁**：**单控制者独占**（已被控时其他端连接即拒）；**本地输入优先**（被控端本地用户正在输入时远程输入被挡，本地空闲才生效）。
- 协议要扩：握手/版本协商、状态增量（grid delta / block 事件 / 布局变更）、操作指令（按键/命令/窗口管理）、心跳、配对。

### part1（后端）已落地的关键设计决策（开工前设计评审 + 实现后 code review 验证）

- **配对码：服务端生成 → 仅发被控端展示 → 控制端人工转述输入 → 服务端校验**。因「起步全走中继」服务端本就明文经手所有终端数据，对它藏码无收益；服务端集中校验能原子处理独占、无客户端互验竞态。`SubmitPairing` 由服务端校验「提交者 == 发起请求的鉴权 device_id」，杜绝抢答/重放/跨用户。
- **控制面 / 数据面分离**：服务端只解释控制消息；数据面 `Relay` 载荷 = 不透明 `serde_json::Value`，**服务端原样盲转不反序列化** → part2/3 给 `RemoteFrame` 加状态增量/Action 变体时**服务端零改动**（前向兼容铁律）。
- **Hub 单锁**：`Mutex<HubState>` 一把锁包 peers/sessions/pending，每操作单锁原子完成、临界区不跨 await，根除三锁死锁/TOCTOU。投递走各连接 mpsc（同步 send）。
- **鉴权走 `Authorization: Bearer` 头**（复用 REST 的 `AuthUser` 提取器），不走 query，避免反代日志泄漏 token。
- **同 device 重连驱逐 + conn_id 守卫**：旧连接被踢且其断开清理不误删新连接。
- **机器可读原因枚举**（`DenyReason`/`PairingFailReason`/`EndReason`）供客户端本地化。配对 attempts=5 + TTL 120s + 后台 GC。
- **未采纳**（超 part1/过度设计）：ws_sessions 表、Relay 重传队列/seq、会话三态 FSM（用 remove 返回 Option 的幂等即可）、token 连接中途吊销（与 M5.1 一致延后到对外硬化）。
- **part2 客户端约定**：消息全走 WS **Text 帧**（serde_json）；客户端用 `tungstenite`（同步）+ rustls/native-tls；升级请求带 `Authorization: Bearer <token>` 头。
