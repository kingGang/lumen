  - 进库：`docker exec -it lumen-postgres psql -U lumen_user -d lumen`
- **server**：`server/lumen-server`，默认监听 `0.0.0.0:8787`，连接串默认指 5544。起服务：`./target/debug/lumen-server.exe`（常驻）。健康检查 `GET /api/v1/health`。
  - ⚠️ 本机有 **HTTP 代理**（`ALL_PROXY=http://127.0.0.1:7897`）：`Invoke-RestMethod` 测本地必须加 `-NoProxy`，否则 502。客户端 ureq 不读环境代理、直连，不受影响。
- **两机互联**：本机局域网 IP `192.168.1.85`。另一台客户端在 **设置→网络→服务器** 填 `192.168.1.85:8787`（自动补 http://）。**机器 A 防火墙需放行 TCP 8787 入站**（海风哥手动跑管理员 PowerShell：删 `lumen-server` 的 Block 规则 + `New-NetFirewallRule ... -LocalPort 8787 -Action Allow -Profile Private,Domain`）。

## 本机工程坑（重要）

- **rustc 1.96 随机崩溃** `STATUS_ACCESS_VIOLATION`（大 crate codegen）：一律用 `CARGO_INCREMENTAL=0` + **重试**绕过。
- 客户端在跑时 `lumen.exe` 被占用、`cargo build` 链接失败：用 `cargo check -p lumen-app` 验证，或让海风哥先关客户端。
- **sqlx 弃用**：sqlx-postgres 0.8.6 在 rustc 1.96 触发编译器 ICE，server 改用 **tokio-postgres + deadpool**。
- **clippy 全局 deny** `cast_*`（用 `try_from` 不用 `as`）+ 禁 `unwrap()`。
- 协议/server clippy 已零警告；客户端全量 clippy（egui/wgpu）太重+flaky，新代码面小、靠 build+test 把关。

## M5.3 终端远程（下一步，海风哥已拍板）

- **形态**：选中在线设备 → 控制端**镜像被控端整个编辑器实例**（会话 tab/目录树/终端），同 Claude Code 远程会话。**状态增量同步 + 操作指令（非画面流）**——见设计 §5.2。
- **通道**：此时才建 **TLS WebSocket 长连接经 lumen-server 中继**（M5.2 只用 HTTP 心跳）。P2P 留后续。
- **连接授权**：首次连接 **9 位数字配对码**；被控端醒目「正在被远程控制」指示 + 一键断开。
- **并发/仲裁**：**单控制者独占**（已被控时其他端连接即拒）；**本地输入优先**（被控端本地用户正在输入时远程输入被挡，本地空闲才生效）。
- 协议要扩：握手/版本协商、状态增量（grid delta / block 事件 / 布局变更）、操作指令（按键/命令/窗口管理）、心跳、配对。
