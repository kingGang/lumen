# Lumen「现代输入编辑器」设计文档

> 状态：定稿草案（2026-06-10，三视角方案竞标 + 评审裁决综合产出）
> 前置：M1 终端底座、M2 Blocks UI 已完成。本文档原对应里程碑 M3；同日「应用外壳」插为新 M3 后，**本里程碑顺延为 M4，文中 M3.x 分期编号请读作 M4.x、M4 远程控制请读作 M5**，其余内容不变。
> 总基调：**Warp 式输入接管，shell 侧近零改动**——shell 保留「画提示符 + 收整行 + 报边界/退出码」三职，编辑/高亮/补全/历史全部上移到 Lumen 侧纯状态机；所有操作协议化为可序列化 Action 汇入唯一 dispatch（远程控制同入口）；**直通态就是回退路径本身**。

## 1. 总体架构与数据流（单向）

```
winit 按键/IME/鼠标 ──┐
                      ├─ keymap（静态表：键位+模式列+守卫 → Action）──┐
M4 远程指令(protocol) ─┘                                            ▼
                                            dispatch(Action) ← 唯一状态变更入口
                                                   │ 返回 Vec<StateEvent>
        ┌──────────────────┬───────────────────────┼─────────────────────┐
        ▼                  ▼                       ▼                     ▼
 lumen-editor::apply  ComposerAction          TermAction          StateEvent 消费方
 纯文档状态机          提交/历史/补全           语义键→VT 编码        渲染/历史库/状态条
        │                  │                  →pty.write            （M4: 网络增量）
        └── EditorView/ComposerView（只读视图）──▶ lumen-renderer
                                                  （grid + footer + overlay 同一 render pass）
```

三条铁律：

1. **一切状态变更走 `dispatch(Action)`**。winit handler 只做「事件 → keymap → Action」翻译，不直接碰 editor/pty/term 状态；M4 远程消息反序列化为同一 `Action` 进同一入口。
2. **lumen-editor 是独立纯状态机 crate**，零 winit/wgpu/pty 依赖，`apply()` 是唯一可变入口，同一 Action 序列产生确定性终态（replay 测试，M4 远程一致性地基）。
3. **渲染只消费只读视图**；编辑器重绘直接 `request_redraw()`，绝不挂 PTY 的 debounce/ESU 节拍（main.rs:424-429），并遵守 ControlFlow 复位约定（main.rs:447-452）。

## 2. 输入模式机（四态，纯推导）

模式是**推导值**，由纯函数从终端状态单点求值，按键处理与每批 PTY 数据消化后实时计算，**禁止任何地方缓存模式副本**（防状态漂移）：

```rust
pub enum InputMode { Compose, Running, AltScreen, Fallback }

fn input_mode(term: &Term) -> InputMode {
    if term.blocks().is_empty()      { return InputMode::Fallback }  // 从未见 133 标记
    if term.is_alt_screen()          { return InputMode::AltScreen } // term.rs:146
    match term.blocks().last() {
        // B 已到、C 未到 = PSReadLine 正在等输入（integration.ps1 在 prompt 末发 B）
        Some(b) if b.cmd_line.is_some() && b.output_line.is_none() => InputMode::Compose,
        _ => InputMode::Running,
    }
}
```

| 模式 | 输入路由 | 输入区形态 |
|---|---|---|
| **Compose** | 按键 → keymap → EditAction，本地编辑 | 完整 Composer 卡片 |
| **Running**（C 后 D 前：y/n 确认、REPL、密码） | **逐键直通 PTY**，不做本地缓冲 | 坍缩为等高状态条（运行中/耗时/「直通中」提示） |
| **AltScreen**（vim/htop/codex） | 完全直通（含 IME、Ctrl+C/V） | 整体隐藏，grid 收回全高 |
| **Fallback**（注入失败/SSH 进无 integration 环境） | 永久直通 = M2 现状 | 隐藏 + 「shell 集成未生效」提示条 |

- Fallback 自然涌现：启动初期 shell 加载 profile 时 blocks 为空 → 直通；第一个 prompt 的 A/B 到达自动进 Compose。注入写出失败（main.rs:79-83 兜底路径）则永不出现标记，永久 Fallback。**不做任何「猜提示符」启发式。**
- 嵌套 shell（pwsh 套 pwsh/cmd/bash/ssh）：内层无 integration 不发标记，外层块停在运行态 → 持续 Running 直通，正确退化；内层退出后外层 prompt 重现（D+A+B），Composer 自动回归。
- **手动逃生**：`Ctrl+Shift+E` 运行时切换「经典直通模式」（强制 Fallback），状态条显示当前模式，误判永远可见、可逃。
- **草稿保全**：模式切换不清编辑器缓冲；vim 退出/命令结束回到 Compose 时草稿原状恢复。

## 3. 输入区与 shell 协作协议

### 3.1 职责划分（拍板）

| 职责 | 归属 | 理由 |
|---|---|---|
| 提示符绘制 | **shell**（starship/posh-git 无损） | cwd/git 本就是 shell 算的；A..B 区间天然是块头；不做 Warp 式 PS1 预渲染捕获 |
| 行编辑/高亮/补全/历史 | **Lumen**（lumen-editor + app 控制器） | M3 主体 |
| 整行接收 + 回显 + 发 C | **PSReadLine（驯化保留）** | C 标记靠包装它发出（integration.ps1:27-34）；bracketed paste 多行接收靠它；**不 Remove-Module** |
| 命令边界/退出码 | 现有 OSC 133 链路 | 零改动（integration.ps1:23 → term.rs:511） |

### 3.2 命令提交协议（时序）

1. **门控**：仅 Compose 态接受 `ComposerAction::Submit`；dispatch 内**二次复核** `input_mode()`，已非 Compose 则拒绝、缓冲保留、状态条提示（不静默吞）。
2. **取文编码**：单行 → `text + "\r"`；多行 → `ESC[200~ + text(换行规整为\r) + ESC[201~ + "\r"`（bracketed paste，查询 `term.bracketed_paste()`，term.rs:159；PSReadLine 整体接收不触发 `>>` 续行）。
3. **写入**：`scroll_to_bottom()` → 一次 `pty.write()`（写线程化已就绪，lumen-pty，天然非阻塞）。
4. **本地记账**：立即清空编辑器缓冲、命令入历史库、记录 `pending_submit { text, block_id, submitted_at }`——提交不创建新块，C/D 都落在当前等待输入块上，命令文本权威来源就是本地 `text`（不从 grid 抓回显）。
5. **回显落块**：PSReadLine 在 B 后回显整行（恰好成为块内命令行记录，Warp 同款）→ 回车触发 `OSC 133;C` → 模式机自动切 Running。
6. **退出码回流**（零改动）：下一 prompt 发 `D;<ec>` → `exit_code` 入块（term.rs:511-514）→ StateEvent「块闭合」→ 更新状态条（✓/✗ + 耗时 + 非零时红色角标，任意按键消除）并写回历史库。该事件即 M4 状态增量的一部分。

### 3.3 integration.ps1 增量（P0 即做）

```powershell
# ① PSReadLine 驯化（每条独立 try/catch —— Windows PowerShell 5.1 自带版无 PredictionSource）
try { Set-PSReadLineOption -PredictionSource None } catch {}
try { Set-PSReadLineOption -BellStyle None } catch {}
# ② cwd 上报：prompt 包装内追加 OSC 9;9;<cwd>（路径补全与历史库 cwd 字段的数据源）
# ③ M3.2 起：ReadLine 包装在发 C 之前发 OSC 633;E;<base64(cmdline)> —— 权威命令文本
```

- **633;E 用途**：①对账——与 `pending_submit.text` 不一致（PSReadLine 历史展开/用户在直通态手敲）时以 shell 上报为准；②Fallback/直通态输入的命令文本来源。首期不依赖它。
- **ConPTY 透传**：M2 已证 133 输出向可穿透；633;E 同为输出向 OSC，M3.1 第一周顺手实测，不通过则降级为 OSC 133 私有参数位携带 base64（已验证通道）。

## 5. lumen-editor crate（纯状态机）

```
crates/lumen-editor/        # 零 winit/wgpu/pty 依赖，100% 可单测
├── action.rs    # EditAction（serde derive，第一天可序列化）
├── document.rs  # 行存储文档模型
├── cursor.rs    # Position/Selection/Motion（grapheme 移动）
├── undo.rs      # undo/redo 栈（连续打字合并为一组）
└── highlight.rs # Highlighter trait + PowerShell tokenizer（M3.2）
```

```rust
pub struct Editor {
    lines: Vec<String>,            // 行数组；命令行 <10KB，rope 是过度设计；接口不暴露存储形态
    cursor: Position,              // { line: usize, byte: usize } 字节偏移，移动按 grapheme
    anchor: Option<Position>,      // 选区锚点（== cursor 即纯光标）
    undo: UndoStack,
    abandoned: Option<String>,     // Ctrl+C 放弃稿
    revision: u64,                 // 单调递增：渲染缓存失效判据 + M4 增量同步版本锚点
}

#[derive(Serialize, Deserialize)]
pub enum EditAction {
    InsertText(String), InsertNewline,
    DeleteBackward, DeleteForward, DeleteWordBackward,
    Move { motion: Motion, extend: bool }, SetSelection(Selection), SelectAll,
    SetText(String),               // 历史/补全整体替换入口
    Undo, Redo, Clear,
}
pub enum Motion { GraphemeLeft, GraphemeRight, WordLeft, WordRight,
                  LineStart, LineEnd, Up, Down, DocStart, DocEnd }

impl Editor {
    pub fn apply(&mut self, a: &EditAction) -> EditOutcome;  // 唯一可变入口
    pub fn view(&self) -> EditorView<'_>;                    // 只读视图：文本+光标+选区+spans
    pub fn needs_continuation(&self) -> bool;                // M3.2：未闭合引号/括号、行尾 ` 或 |
}
```

**不变量**：除 `apply()` 外无公开可变入口（字段全私有，类型层强制）；replay 测试保证同一 Action 序列 → 确定性终态。

### 现有四 crate 改动面

| crate | 改动 | 量级 |
|---|---|---|
| lumen-term | `shell_waiting_input()` 查询（封装「最后块有 B 无 C」）；OSC 9;9 → `cwd` 字段；M3.2: OSC 633;E → `Block.cmd_text`；`Block.cmd_text` 可由 app 回填本地提交文本 | 小 |
| lumen-pty | 零改动 | 零 |

## 6. Action 总线与 M4 预留

```rust
// lumen-app/src/action.rs（全 serde derive，带版本注释）
pub enum Action {
    Edit(EditAction),                  // → editor.apply()
    Composer(ComposerAction),          // Submit / HistoryPrev / HistoryNext / HistorySearch{..}
                                       // / Completion{Open,Next,Prev,Accept,Dismiss} / CancelLine
    Term(TermAction),                  // SendKey(KeyStroke) 语义键 | SendText(String) | Interrupt
                                       // | Paste(String) | Scroll(..) | JumpBlock(..) | CopySelection
}

fn dispatch(&mut self, action: Action) -> Vec<StateEvent>;   // 唯一执行入口
```

1. **编码下沉**：input.rs 的 VT 编码从事件处理内联改为 dispatch 内服务——`TermAction::SendKey` 在 dispatch 时查终端模式（bracketed paste / win32-input / alt screen）现场编码写 PTY。**本地与远程同一条路径**：M4 远端发语义级 SendKey，被控端按自己的模式编码，终端模式状态不需要跨网同步。
2. **弹窗状态也走 Action**：补全/历史面板的开闭、选中项、过滤词建模为 app 内控制器状态机，变更经 `ComposerAction`——否则 M4 远程端看不到弹窗、无法远程接受补全。
3. **观察者出口**：dispatch 返回 `Vec<StateEvent>`（编辑器 revision 变更、模式切换、块闭合、提交、弹窗状态）——M3 驱动状态条/历史库，M4 即状态增量同步的事件源。**入口收指令，出口吐增量，双向备好。**
4. **纪律**：凡绕过 dispatch 直接改状态的代码，code review 一律打回。

## 7. 渲染方案

### 7.1 footer 输入区（零新 GPU 管线）

- 同一 render pass：卡片背景（不透明）/上边框 1px/竖条光标（~2px）/选区高亮全部用现有 `RectInstance`（alpha 混合已开）；文本用**独立 glyphon TextBuffer 池**（不混入 row_segs 行缓存，哈希语义不耦合；行数少，每帧重排无性能问题）；高亮 spans 走 `set_rich_text`（同 lib.rs:443-449 的 cell_attrs 模式）。
- **几何**：`grid_size()`（lib.rs:155-161）改为 `rows = (usable_h - footer_h) / cell_h`；`cell_at()` 排除 footer 区域。grid 扣高后终端 TextArea 天然不进入 footer，无遮盖问题。
- **防 resize 风暴（关键决策）**：footer **常驻等高**（min 高度 = 1 行 + 内边距），Compose↔Running 状态条切换**不改高度** → 不触发 `term.resize`/`pty.resize`。只有多行编辑增高才走现有 resize 链（main.rs:481-485），**100ms 防抖**，上限 1/3 窗高（超出内部滚动，`TextBounds` 裁剪现成）。AltScreen 隐藏 footer、grid 收回全高（一进一出各一次 resize，与窗口 resize 同路）。
- **光标定位**：首期等宽 + 宽字符占 2 列沿用网格规则（与终端一致）；M3.2 随鼠标点击定位一并换 cosmic-text `layout_runs` 精确像素命中（CJK/连字正确，输入区不受 PTY 网格约束）。
- 圆角：rect.wgsl 无圆角能力，首期直角卡片；SDF 圆角列停车场。

### 7.2 overlay 浮层阶段（M3.3 新建，体验倒推出的硬需求）

现有「全部矩形→全部文字」次序（lib.rs:524-527）下，浮层背景混入主矩形批会被后画的终端文字穿透。方案：

- RectRenderer 增加第二实例批 `overlay_rects`；新建第二个 `TextRenderer`（**共享 FontSystem/TextAtlas/Viewport**，串行 prepare）；
- 同一 render pass 内 draw 次序：`主矩形 → 主文字 → overlay 矩形 → overlay 文字`，无新管线、无新 pass；
- 补全弹窗/历史面板均画在此层，背景不透明主题色 + 1px 边框。

### 7.3 IME（E6）

- Compose 态：`Ime::Commit` → `EditAction::InsertText`；`Ime::Preedit` 进**视图层**（下划线样式内嵌光标处，不进文档模型、不参与 undo）；光标移动时调 `set_ime_cursor_area`（候选框跟随）。
- Running/AltScreen/Fallback：维持现状（Commit 直写 PTY，main.rs:658）。

### 7.4 节拍纪律（E1）

编辑器变更直接 `request_redraw()`，不触碰 `redraw_at` 等 PTY debounce 字段，不引入新的 WaitUntil 残留（main.rs:447-452 约定）；打字延迟与输出流彻底解耦，埋点沿用 `last_key_at`。

## 8. 高亮 / 补全 / 历史

| 能力 | 来源 | 分期 |
|---|---|---|
| **历史（↑/↓）** | 自建 `%LOCALAPPDATA%/Lumen/history.jsonl`（追加写，启动加载内存索引），记 `{text, cwd, exit_code, duration, ts}`；首启一次性导入 `(Get-PSReadLineOption).HistorySavePath` 种子 | **M3.1** |
| **语法高亮** | 自写 PowerShell 轻量 tokenizer（~300 行：命令名/参数 `-Foo`/字符串含未闭合/变量 `$x`/管道与操作符/注释 → `Attrs` spans）；`Highlighter` trait 隔离，日后可换 tree-sitter；`needs_continuation()` 复用同一 tokenizer | **M3.2** |
| **历史搜索（Ctrl+R）** | overlay 面板：模糊过滤（子序列匹配 + 频率/近因加权）、命中高亮、cwd/✗✓ 列、↑↓ 预览、Enter 填入输入区不直接执行、Esc 关闭 | **M3.3** |
| **ghost text** | 历史最佳前缀匹配灰字，→/End 接受（shell 侧 prediction 已驯化关闭，必须本地补回，否则净倒退） | **M3.3** |
| **补全（Tab）** | sidecar 隐藏 pwsh 进程（独立 runspace，stdin/stdout JSON 行协议）调 `[CommandCompletion]::CompleteInput()`（VS Code PSES 同款）；cwd 经 OSC 9;9 同步 `Set-Location`；文件路径补全本地直读（最快路径，首批）；Tab 显式触发；20ms 未返回先显历史前缀项，结果到达合并刷新——**弹窗永不卡打字**；sidecar 崩溃自动重启、Tab 静默降级 | **M3.4** |

已知局限（文档化）：sidecar 拿不到交互会话内的临时函数/变量/别名（Warp 同样拿不到）；**不向用户会话发隐藏命令**（回显隐藏脆、污染会话史）。fig specs 描述列、PSReadLine 历史回写开关列停车场。

## 9. 交互程序场景专章

- **alt screen（vim/htop/less/codex TUI）**：`is_alt_screen()` 翻转瞬间——进入：footer 隐藏、grid resize 回全高、一切按键与 IME 直通（含 Ctrl+C/V/Tab；Shift+PgUp 翻屏与 Ctrl+↑/↓ 现状放行保留）；退出：footer 复现、**草稿原状恢复**。codex 的 ESU 直渲快路（main.rs:400-419）不受影响——Composer 重绘与 PTY 节拍完全隔离。
- **主屏交互（python/node REPL、y/n 确认、`Read-Host`、ssh 密码）**：处于 Running（块有 C 无 D），逐键直通 stdin；回显与密码不回显由对端程序决定，行为与经典终端完全一致。**不做「运行态本地缓冲再转发」的聪明设计**——逐键直通是唯一不会出错的语义。footer 显示「直通中」状态条提示按键去向。
- **命令秒退**（C 与 D 几乎同时）：模式机在同批数据内收敛，等高 footer 保证零布局抖动。
- **嵌套 shell**：内层无 integration → 外层块停运行态 → 持续直通正确退化；子 shell 注入（Warpify 式 DCS）列 M3 之后。
- **主屏自画菜单等方向键的极少数程序**：同属 Running 直通，已覆盖。

## 10. 实施分期与验收

### P0 地基（0.5~1 天，随 M3.1 启动）

- integration.ps1：PSReadLine 驯化（逐条 try/catch）+ OSC 9;9 cwd 上报；lumen-term：`shell_waiting_input()` + cwd 解析 + 单测；ConPTY 输出向 OSC 633 透传实测（一次性专项）。

- **回退**：纯增量，revert 即回。

### M3.1 底座（~2 周）

- 范围：lumen-editor crate（多行模型 + 光标/选区 + undo + Action/Outcome + replay 测试）；四态模式机；keymap 静态表替换 if-else 链（语义表逐条单测）；dispatch + StateEvent 骨架（编码下沉）；footer 渲染（纯文本单色、常驻等高）+ grid 扣高 + 增高防抖；提交全链路（单行 + 多行 bracketed paste + 二次门控 + pending 对账 + 退出码角标）；历史库写入 + ↑/↓ 导航 + 种子导入；IME Commit 分流 + `set_ime_cursor_area`；alt/运行态让位；Fallback 降级 + Ctrl+Shift+E 手动直通；feature flag `input-editor` 可整体编译剔除。
- **bracketed paste 多行提交在第一周专项实测**（ConPTY 对 200~/201~ 透传验证；失败 fallback：反引号续行拼单行）。
- **验收**：E1/E2/E7/E8/E9 成立——①日常单行命令流（中文、粘贴、↑历史）全程在输入区完成；②多行 Shift+Enter 编辑 + 整体提交正确执行不触发 `>>`；③vim/htop/codex 进出让位无残影、运行中 Ctrl+C 可中断、python REPL 可交互、y/n 确认正常；④starship/zoxide 组合无损；⑤删 integration 脚本后自动退回 M2 行为；⑥打字延迟不受持续输出影响（埋点对比）；⑦keymap 与 replay 单测全绿；clippy -D warnings。
- **回退**：Ctrl+Shift+E 运行时一键回经典模式；feature flag 整体剔除。

### M3.2 编辑体验（~1.5 周）

- 范围：IME Preedit 内嵌渲染；鼠标点击定位/拖选/双击选词（layout_runs 像素命中）；剪贴板键位补全（Shift+方向、Ctrl+A/X、Ctrl+Backspace）；语法高亮 tokenizer + Theme token 色板；续行检测（`needs_continuation()`，未闭合引号/管道时 Enter 自动换行）；OSC 633;E 权威文本 + 对账（透传不通则 133 私参 base64）；增高抖动打磨。
- **验收**：E3/E6 成立——中文组字内嵌预览、候选框跟随；粘贴 10 行脚本一次执行；五类 token 着色正确不闪烁、高亮可开关；633;E 对账后块命令文本不依赖回显抓取；增高时 htop 不闪烁。

### M3.3 历史搜索（~1 周）

- 范围：overlay 渲染阶段（第二矩形批 + 第二 TextRenderer 共享 atlas）；Ctrl+R 面板（模糊匹配 + 命中高亮 + cwd/✗✓ 列）；ghost text；↑ 非空前缀过滤；可选项：Running 态 Ctrl+Shift+E 离线起草。
- **验收**：E4 成立——overlay 正确盖住终端文字、atlas 无闪烁丢字形；万条历史过滤 <10ms；ghost text →/End 接受；Esc 逐层关闭。

### M3.4 补全与协议化收尾（~2 周）

- 范围：sidecar pwsh 补全服务（JSON 行协议 + cwd 同步 + 崩溃自愈）；文件路径本地补全；补全弹窗（复用 overlay，单候选内联、多候选 ↑↓ 选择）；Action serde 全量测试 + 录制/回放；键表外置雏形评估；性能埋点；架构设计.md M3 章节落档。
- **验收**：E5 成立——`cd <Tab>`、`git ch<Tab>`、`.\scr<Tab>` 出正确候选且 cwd 跟随；sidecar 崩溃 Tab 静默降级不卡 UI、自动重启；弹窗期间打字零卡顿；Action 序列可录制/回放（M4 lumen-protocol 抽取就绪）。

> 总周期约 6.5~7 周。每期收尾跑全量体验基准回归（含 Fallback 路径），`cargo clippy -- -D warnings` 零告警。**M3 成功标准：M3.1 结束起，开发者自己日常工作切到输入区模式且不想切回去。**
