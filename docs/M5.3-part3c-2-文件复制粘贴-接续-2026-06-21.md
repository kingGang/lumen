## 4. 关键根因 / 技术决策（新会话必读）

- **egui 0.34 context_menu deprecated** → 用 `egui::Popup::context_menu(&resp)`。
- **egui auto-id 在 `ScrollArea + horizontal + push_id` 嵌套下相撞** → 远程树（自绘）的点击 / 双击 / 右键菜单
  **必须用 `ui.interact(rect, egui::Id::new(("tag", row.id)), Sense::click())` 显式 id**，不能依赖
  `selectable_label` 的 auto-id（这是「选不中 / 复制第一个 / 快捷键无效」的总根因）。`selectable_label` 现仅
  用于画选中高亮。
- **egui-winit 吞 Ctrl+C/V**：`on_keyboard_input` 里 `is_paste_command` 命中时 `clipboard.get()` 读**文本**，
  文件剪贴板（CF_HDROP）无文本 → 既不产生 `Event::Paste`、又 `return` 吞掉 V 键事件。故文件树 Ctrl+C/V
  **egui input 层根本拦不到，必须在 winit 原始事件层（main KeyboardInput）拦**。门控不能用 `terminal_focused`
  （点终端才设 true，点文件树不改），改用 `filetree_hovered`（鼠标在面板内，与右键菜单一致）。
- **被控端文件服务 worker 读完必须 nudge** 唤醒主线程（`request_repaint` + `PtyWake`），否则空闲被控端结果
  卡到下个偶发事件（ListDir 慢、可能下载也受影响 → 见下）。
- **系统文件剪贴板**：CF_HDROP（`DROPFILES` 头 + 双 null 宽字符路径列表）。features：
  `Win32_System_DataExchange` / `Win32_System_Ole` / `Win32_System_Memory`（`Win32_UI_Shell` 已有）。
