# Lumen 架构设计

> 一个类 Warp 的现代 GPU 加速终端。Rust 编写，Windows 优先，架构预留跨平台与 Blocks 扩展。

## 1. 产品定位

Lumen 是一个面向开发者的现代终端模拟器，对标 Warp 的核心体验：

- **高性能 GPU 渲染**：wgpu 驱动，目标 144fps 流畅滚动，大输出不卡顿。
- **块式输出（Blocks）**：每条命令与其输出构成独立块，可折叠、复制、跳转（M2）。
- **现代输入编辑器**：多行编辑、语法高亮、补全，而非 readline 行编辑（M3）。
- **AI 集成**：自然语言转命令、报错解释（M4，暂缓）。

MVP 路线：**先做纯终端底座**——能日常跑 PowerShell 的高质量终端模拟器，但数据模型从第一天就为 Blocks 设计。

## 2. 技术选型

| 层 | 选型 | 理由 |
|---|---|---|
| 窗口/事件 | winit 0.30 | Rust 桌面事实标准，ApplicationHandler 模型 |
| GPU 渲染 | wgpu | 跨平台（Vulkan/DX12/Metal），Warp 同思路 |
| 文本排版/渲染 | cosmic-text + glyphon | glyphon 提供现成 glyph atlas 与渲染管线，M1 不自造轮子；wgpu 版本跟随 glyphon 锁定 |
| PTY | portable-pty | wezterm 出品，封装 ConPTY 与 unix pty，天然跨平台抽象 |
| VT 解析 | vte | alacritty 出品的 VT100/ANSI 状态机解析器 |

**版本策略**：wgpu 的版本以 glyphon 的依赖为准，避免图形栈版本打架。自研 glyph atlas 渲染（替换 glyphon）列为 M2 之后的性能优化项。

## 3. Crate 划分（cargo workspace）

```
Lumen/
├── Cargo.toml            # workspace，统一依赖版本
├── docs/                 # 设计文档（中文）
└── crates/
    ├── lumen-pty/        # PTY 抽象层
    ├── lumen-term/       # VT 解析 + 终端状态机（核心数据模型）
    ├── lumen-renderer/   # wgpu + glyphon 渲染
    └── lumen-app/        # winit 主程序，组装一切
```

### 3.1 lumen-pty

- `PtySession`：spawn shell（Windows 默认 `pwsh.exe`，回退 `powershell.exe`）、`resize(rows, cols)`、`writer()`。
- 输出读取在独立线程进行，通过 `crossbeam-channel` 发送 `Vec<u8>` 给主循环。
- 依赖 portable-pty，本身不含平台 `cfg`——跨平台差异由 portable-pty 吸收。

### 3.2 lumen-term（核心）

终端状态机，**不依赖任何图形库**，纯数据模型，可独立单测：

- `Cell`：字符 + 前景色 + 背景色 + 样式标志（粗体/斜体/下划线/反显）。
- `Grid`：可视区 cell 矩阵 + scrollback 环形缓冲（默认 10k 行）。
- `Performer`（实现 `vte::Perform`）：
  - `print`：写字符，处理宽字符（东亚全角占 2 列）。
  - `execute`：C0 控制（CR/LF/BS/TAB/BEL）。
  - `csi_dispatch`：光标移动（CUP/CUU/CUD…）、擦除（ED/EL）、SGR 颜色样式、滚动区、模式设置（含备用屏幕 1049）。
  - `osc_dispatch`：窗口标题（OSC 0/2）；**OSC 133 命令边界标记**（A=提示符开始、B=命令开始、C=输出开始、D=命令结束）→ 这是 Blocks 的数据基础，M1 即记录。
- `Block`：`{ id, prompt_start, cmd_start, output_start, end, exit_code }`，以行号区间标记在 scrollback 上。M1 只采集不展示，M2 做折叠/跳转 UI。

### 3.3 lumen-renderer

- wgpu 初始化：`Instance → Surface → Adapter → Device/Queue`，surface 格式优先 sRGB。
- glyphon：`FontSystem`（系统等宽字体，优先 Cascadia Mono/Consolas）+ `TextAtlas` + `TextRenderer`。
- 每帧：将 Grid 可视区转为带颜色的 rich text spans → cosmic-text Buffer 排版 → glyphon 渲染；另用单色矩形管线画背景色块与光标。
- 单元格尺寸由字体 metrics 决定，`resize` 时反推 rows/cols 通知 PTY。

### 3.4 lumen-app

- winit `ApplicationHandler`：`resumed` 创建窗口与渲染器，`window_event` 处理输入/重绘/resize。
- 数据流（单向）：

```
PTY 读线程 ──bytes──▶ channel ──▶ 主循环喂 vte Parser ──▶ Grid 更新 ──▶ request_redraw
键盘事件 ──编码(VT 序列)──▶ PTY writer
```

- 键盘编码：可见字符直接写；Enter=`\r`、Backspace=`\x7f`、方向键=`CSI A/B/C/D`、Ctrl+字母=控制字节。
- PTY channel 用 winit 的 `EventLoopProxy` 唤醒事件循环（user event），避免轮询。

## 4. 里程碑

| 里程碑 | 内容 | 验收 |
|---|---|---|
| **M1 终端底座** | 窗口 + ConPTY + VT 解析 + GPU 渲染，能日常交互跑 PowerShell；OSC 133 块边界已采集 | vim/top 不要求，`ls`、`git log`、交互输入、颜色、resize、滚动正常 |
| M2 Blocks UI | 块折叠/复制/跳转、shell integration 脚本注入、备用屏幕完善 | Warp 式块交互 |
| M3 输入编辑器 | 独立输入区、多行编辑、高亮、补全、历史搜索 | IDE 级输入体验 |
| M4 AI | 自然语言转命令、报错解释（接自建中转站） | — |

## 5. 工程约束

- `cargo clippy -- -D warnings` 零告警；禁 `unwrap()`（库代码用 `Result`/`expect` 带说明）。
- lumen-term 必须保持 100% 无图形依赖，核心逻辑（光标/擦除/SGR/宽字符）配单元测试。
- 文档与注释一律中文；公共 API 带 rustdoc。
