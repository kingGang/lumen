## 1. 背景与定位

Lumen 目前是**纯本地**终端：登录是本地假登录（`profile.rs::mock_login` 只校验邮箱含 `@`、密码非空），无任何网络/账户后端（除热更下载用 `ureq`）。M5 要补上"同账号多设备互见互控"这条主线——区别于 Warp 的单会话共享，Lumen 控制的是**整个远端编辑器实例**（所有标签页 / 窗格 / 目录树 / 输入区），交互形态类向日葵设备列表 + Claude Code 远程会话的接管感。

核心实现路线（沿用 §5.2）：**状态增量同步 + 操作指令，而非画面视频流**。终端状态本质是文本与结构化数据（Grid、Blocks、输入区、布局），被控端把状态增量推给控制端，控制端用**本地渲染管线**绘制——画面无损、延迟低、带宽比视频流小几个量级。控制端的操作编码成 Action 指令发回被控端，走被控端本地输入同等路径执行（M4 已预留统一 Action 总线，见 `docs/输入编辑器设计.md`）。

## 3. 总体架构

### 3.1 新增组件（沿用 §5.3）

```
Lumen/
├── crates/
│   ├── lumen-protocol/   # 新增：协议消息（serde 结构体），客户端/服务端共享，带版本字段
│   └── …                  # 现有五 crate 不变
└── server/
    └── lumen-server/     # 新增：Rust + tokio + axum 云端服务，单二进制部署自有 VPS
```

- **lumen-server**：HTTPS REST 负责注册/登录/设备列表/个人数据同步；WebSocket 长连接负责在线状态心跳 + 控制流量与文件流量中继。
- **lumen-protocol**：握手与版本协商、状态增量（grid delta / block 事件 / 布局变更）、操作指令（按键 / 命令执行 / 窗口管理）、心跳、文件传输块。从第一天带版本字段，workspace 内两端共享，类型不漂移。

### 3.3 部署与测试环境

- **服务端（lumen-server，双平台编译）**：**发布**编 **Linux** 版部署到云服务器（axum 单二进制 + Postgres + Caddy 反代 / Let's Encrypt 自动续期，已有域名、国内需 ICP 备案）；**本地测试**编 **Windows** 版直接跑（连本地 Postgres，可用 Docker 起）。因此 lumen-server 只用**纯跨平台 crate**（tokio/axum/sqlx 等），保证 Windows + Linux 两平台都能编、行为一致。
- **客户端（Lumen）**：始终 **Windows**——被控端与控制端两端均为 Windows，PTY 均为 ConPTY、路径均为 Windows 风格，**客户端本身不做跨平台**（不涉及 Mac）。
- **共享协议**：`lumen-protocol` 纯 serde、**零平台依赖**，Win 客户端 / Win 测试 server / Linux 生产 server 三方通用。
- **测试**：功能测试在 **Windows** 上进行（含本地 Windows 版 server）；同机起两个客户端实例需绕过单实例检测（F8），或用两台 Windows 机。
- **构建隔离 + CI**：lumen-server 构建链绝不碰 Windows-only 客户端 crate（`lumen-app`/`lumen-pty`/`lumen-renderer`，含 windows-sys/ConPTY/wgpu）——用 server 独立 Cargo 或 `cargo build -p` / per-target 依赖隔离。CI 三目标：Linux 编 server（发布）、Windows 编 server（测试）、Windows 编客户端。

### 3.2 客户端关键接入点（探查结论）

数据 I/O 收口已经很干净，远程数据源可**非侵入式接入**：

- **PTY 收口**：`crates/lumen-pty/src/lib.rs` —— `PtyEvent::Data(Vec<u8>)`（唯一读出口）、`pty.write(bytes)`（唯一写入口，:179）。把 `PtySession` 抽象成 trait，新增 `RemotePtySession` 实现，本地/远程对称替换。
- **Session 写收口**：`crates/lumen-app/src/session.rs::write_user_input`（:426）已是唯一用户输入入口。
- **Action 总线**：M4 已让本地输入与（未来）远程指令汇聚到统一 dispatch，控制端指令复用此路径。
- **文件树数据源**：`crates/lumen-app/src/shell/filetree.rs::read_dir_worker`（:1590）紧耦合 `std::fs`，需抽出 `FileSource` trait（本地 fs / 远程）。
- **账户/档案**：`crates/lumen-app/src/profile.rs`（`Profile` 仅 email/display_name/logged_in_at）需扩展 token + device_id；`shell/login_ui.rs` 的假登录换真 REST 调用。

## 4. 子里程碑拆分

### M5.1 账户系统 + 设备设置数据同步

合并需求池 **F9 用户数据云同步**（"同账号多设备自动同步"）。

- **服务端（lumen-server 骨架）**：注册 / 登录（密码 argon2 哈希、签发设备短期 token）、账户库、个人数据库（per-account）。HTTPS REST。
- **客户端**：
  - 真登录替换 `mock_login`；`Profile` 扩展 `device_id`（生成并持久化）、token（短期、不长存）。
  - 设置数据同步，按 **F9 清单**（`docs/需求池.md` §F9）落地：
    - ✅ 同步：命令历史 `history.jsonl`（多设备按 `text+ts` 去重合并，cwd 跨机仅供展示）、UI 偏好 `settings.json` 字段（主题/语言/字号/经典模式/面板显隐等）、自定义主题、补全/历史用户配置。
    - ⚠️ 斟酌/部分：背景图（开关/不透明度可同步，本地图片路径不可）、布局像素、会话结构（cwd 机器相关）。
    - ❌ 不同步：窗口尺寸位置、运行态。
  - 同步机制：增量（`ts`/版本号 watermark）、字段级 last-write-wins / 合并、**离线优先**（本地为主、联网回灌、断网照常用）、登出本地缓存清理（策略待定）、命令历史含敏感信息可考虑端到端加密。
- **协议**：lumen-protocol 起步——REST DTO + 同步消息，带版本字段。
- **交付/验收**：两台机同账号登录，A 机敲的命令历史 / 改的主题，B 机能自动同步出现。

### M5.2 设备在线

- **服务端**：WebSocket 长连接 + 心跳；维护每账户的设备在线状态与设备列表（含离线设备）。
- **客户端 UI**：
  - 顶栏新增「本地 / 远程」两 tab（`shell/topbar.rs`，HEIGHT 34，左端三视图按钮组旁插一对 tab，`TopbarOutput` 加切换信号）。
  - 远程设备列表作为**会话列表左侧第三列** `Panel::left("lumen_remote_list")`（`shell/mod.rs` 侧栏面板区 :514 前插入），settings 加 `remote_list_width` / 显隐字段；**仅"远程"tab 显示**。
  - 列表项复用会话侧栏渲染范式（`mod.rs::TabItem` :33 / `sidebar_ui` :1352）：在线置顶、离线置底，离线项灰显标"不可连接"。
  - 右键菜单复用 `context_menu`（:1545）：**重命名**（复用 :1434 重命名机制，默认显示电脑名）+ **删除**。
- **交付/验收**：远程 tab 能看到同账号所有设备，在线/离线分区正确，可改名、可删除。

### M5.3 终端远程（核心）

- **协议**：lumen-protocol 扩展握手/版本协商、状态增量（grid delta、block 事件、布局/标签页变更）、操作指令（按键、命令执行、窗口/标签页管理）、心跳。
- **连接授权（9 位数字配对码）**：控制端选中在线设备发起连接 → 被控端弹 **9 位数字配对码** → 控制端输入正确配对码方可建立控制会话。
- **控制独占**：一台被控端同一时刻只接受一个控制端；已被控时其他端发起连接**直接拒绝**（不排队、不抢占）。
- **输入仲裁（本地优先）**：被控端本地用户正在输入文字期间，远程控制端的输入被挡住；本地用户停止输入后远程输入才生效。需在被控端定义"本地正在输入"判定（如输入区有未提交内容、或最近一段时间内有本地按键）。
- **被控端**：醒目的「正在被远程控制」指示 + 一键断开；把整编辑器状态增量广播给控制端。
- **控制端**：选中设备后整个工作区镜像该设备（会话 tab / 目录树 / 终端），用本地渲染管线绘制；本地操作编码为 Action 指令经中继发回被控端，走其本地 dispatch 执行。
- **数据通道**：TLS WebSocket 经 lumen-server 中继。
- **交付/验收**：A 机控制 B 机，在 A 上新建标签页/敲命令，B 实际执行且 A 实时看到回显，延迟可用；B 有被控提示且能一键断开。

### M5.4 文件传输

- **协议**：远程目录列表、文件块传输（带进度）、取消。
- **目录树**：抽 `FileSource` trait（本地 fs / 远程），`read_dir_worker` 多态化；远程目录走懒加载 + 后台读取（复用现有 mpsc + 代次号去重机制 `filetree.rs` :376/:1552）。
- **打开（双击）**：远程文件 → 拉到**本地临时夹** → `open_with_default`（:1850）。`paths.rs` 新增 `temp_dir()` + **LRU 容量淘汰**（默认上限 **500M**、可在设置中调整，超上限顶替最久文件）；settings 加临时夹容量字段、设置界面加对应项。
- **复制 / 粘贴（远程桌面式）**：复制远程文件**不立即传**，仅记住来源；**粘贴时**才把数据传输到用户指定的目标路径。
- **状态栏进度**：`shell/statusbar.rs` 中间 cwd 区条件渲染（`StatusBarOutput` 加 `transfer_state`，复用现有色板 + painter）：左显文件名（超宽 `…` 截断）、进度条、**右显传输中文件数**；多文件并发**每几秒轮换**显示一个文件状态。
- **异步承接**：复用现有 `std::thread` + mpsc + 100ms 轮询重绘；**新增** progress 通道 + 取消机制（`Arc<AtomicBool>`，参考搜索的 `search_epoch`）。
- **交付/验收**：远程目录树可浏览；双击远程文件能打开（落临时夹、容量淘汰生效）；复制远程文件后粘贴到本地指定路径成功；传输全程状态栏显示文件名+进度+并发计数。

## 6. 安全基线（沿用 §5.4 + 本次新增）

- 全链路 TLS；密码 argon2 哈希存储；设备持短期 token 而非长存凭据。
- 被控端醒目「正在被远程控制」指示 + 一键断开。
- **新增（2026-06-17）**：首次连接需配对码授权，防止同账号下未经确认的接管。
- 命令历史等个人数据同步可考虑端到端加密。
